reflexpr header synopsis

The standard <reflexpr> header implements the interface of the metaobject types generated by the compiler. 

namespace std {

is_metaobject type-trait

The std::is_metaobject type-trait is used to distinguish between metaobject types and all other types. 

template <typename T>
struct is_metaobject
 : integral_constant<bool, implementation-defined >
{ };

template <typename T>
constexpr bool is_metaobject_v = is_metaobject<T>::value;

true if T is a Metaobject false otherwise.

namespace meta

All reflection-related definitions (except for the is_metaobject type trait) are nested in the std::meta namespace: 

namespace meta {

Object

The meta::Object concept is modelled by stateless anonymous types generated by the compiler, which allow access to metadata reflecting specific program declarations. 

template <typename T> concept bool Object
 =  is_metaobject_v<T>;

Object operations

The following operations are applicable to all metaobjects. 

// reflects_same
template <Object T1, Object T2>
struct reflects_same
 : integral_constant<bool, implementation-defined >
{ };

template <Object T1, Object T2>
constexpr bool reflects_same_v
 = reflects_same<T1, T2>::value;


// get_source_file
template <Object T>
struct get_source_file {
	using value_type = const char [N+1];
	static constexpr value_type value = implementation-defined; 
};

template <Object T>
constexpr auto get_source_file_v
 = get_source_file<T>::value;


// get_source_line
template <Object T>
struct get_source_line
 : integral_constant<unsigned, implementation-defined >
{ };

template <Object T>
constexpr auto get_source_line_v
 = get_source_line<T>::value;


// get_source_column
template <Object T>
struct get_source_column
 : integral_constant<unsigned, implementation-defined> 
{ };

template <Object T>
constexpr auto get_source_column_v
 = get_source_column<T>::value;

true if both T1 and T2 reflect the same base-level declaration, false otherwise.

Null-terminated constexpr char array containing the path to the source file, where the base-level entity reflected by T was declared. If T does not have an in-source declaration (for example a built-in type), the value is an empty (null-terminated) string.

The source file line number, where the base-level entity reflected by T was declared. If T does not have an in-source declaration (for example a built-in type), then the value is zero.

The source file column number, where the base-level entity reflected by T was declared. If T does not have an in-source declaration (for example a built-in type), then the value is zero.

ObjectSequence

meta::ObjectSequence is a concept for lazy metaobject sequences. 

template <Object T> concept bool ObjectSequence
 =  implementation-defined;

ObjectSequence operations

Operations applicable to metaobject sequences -- types conforming to the meta::ObjectSequence concept: 

// get_size
template <ObjectSequence T>
struct get_size
 : integral_constant<unsigned, implementation-defined >
{ };

template <ObjectSequence T>
constexpr auto get_size_v
 = get_size<T>::value;


// get_element
template <ObjectSequence T, unsigned I>
struct get_element {
	using type = Object 
};

template <ObjectSequence T, unsigned I>
using get_element_m
 = typename get_element<T, I>::type;


// unpack_sequence
template <ObjectSequence T, template <class ...> class Tpl>
struct unpack_sequence {
	using type = Tpl<Object ... >;
};

template <ObjectSequence T, template <class ...> class Tpl>
using unpack_sequence_t
 = typename unpack_sequence<T, Tpl>::type;

The number of elements in the metaobject sequence T.

The I-th metaobject in the metaobject sequence T.

A pack of all metaobjects in the metaobject sequence T.

Reversible

Metaobjects modelling the meta::Reversible concept can be reverted to the original declaration. This means that for example a meta::Type can be reverted to the base-level type which it reflects. 

template <Object T> concept bool Reversible
 =  implementation-defined;

Named

Metaobjects modelling the meta::Named concept reflect base-level declarations or other entities which can have a name or are referred to by an identifier or a keyword. 

template <Object T> concept bool Named
 =  implementation-defined;

Named operations

Operations applicable to metaobjects conforming to the meta::Named concept: 

// is_anonymous
template <Named T>
struct is_anonymous
 : integral_constant<bool, implementation-defined >
{ };

template <Named T>
constexpr auto is_anonymous_v
 = is_anonymous<T>::value;


// get_base_name
template <Named T>
struct get_base_name {
	using value_type = const char [N+1];
	static constexpr value_type value = implementation-defined; 
};

template <Named T>
constexpr auto get_base_name_v
 = get_base_name<T>::value;


// get_display_name
template <Named T>
struct get_display_name {
	using value_type = const char [N+1];
	static constexpr value_type value = implementation-defined; 
};

template <Named T>
constexpr auto get_display_name_v
 = get_display_name<T>::value;

true if the base-level entity reflected by T is anonymous (like a anonymous namespace or class), false otherwise.

Null-terminated constexpr char array containing the base name of the base-level named declaration reflected by T. If the base-level declaration is anonymous then value is an empty null-teminated char array. The get_base_name operation returns the type, namespace, etc. name without any qualifiers, asterisks, ampersands, angle or square brackets, etc.

Null-terminated constexpr char array containing the display name of the base-level named declaration reflected by T. If the base-level declaration is anonymous then value is an empty null-teminated char array. The name returned by this operation is implementation-defined and it should be as close as possible to the declaration name as spelled in the source.

Alias

Some named base-level declarations can have one or several aliases (for example type, template or namespace aliases). These are reflected by metaobjects modelling the meta::Alias concept. All meta::Aliases also conform to the meta::Named concept. 

template <Object T> concept bool Alias
 =  Named<T>
 && implementation-defined;

Alias operations

Operations applicable to metaobjects conforming to the meta::Alias concept: 

// get_aliased
template <Alias T>
struct get_aliased {
	using type = Named; 
};

template <Alias T>
using get_aliased_m
 = typename get_aliased<T>::type;

The reflection of the original, underlying named declaration aliased by the alias reflected by T.

Typed

Models of the meta::Typed concept reflect base-level declarations with a type (for example variables or constants) 

template <Object T> concept bool Typed
 =  implementation-defined;

Typed operations

Operations applicable to metaobjects conforming to the meta::Typed concept: 

// get_type
template <Typed T>
struct get_type {
	using type = Type; 
};

template <Typed T>
using get_type_m
 = typename get_type<T>::type;

The reflection of the type of the typed declaration reflected by T.

Scope

Base-level scopes, like namespaces, classes, enumerations, etc. are reflected by metaobjects modelling the meta::Scope concept. 

template <Object T> concept bool Scope
 =  implementation-defined;

Namespace

Namespaces are reflected by metaobjects conforming to the meta::Namespace concept. All meta::Namespaces also conform to meta::Named, meta::Scope and meta::ScopeMember. 

template <Object T> concept bool Namespace
 =  Named<T>
 && Scope<T>
 && ScopeMember<T>
 && implementation-defined;

GlobalScope

The global scope is reflected by metaobjects modelling the meta::GlobalScope concept. 

template <Object T> concept bool GlobalScope
 =  Namespace<T>
 && implementation-defined;

Type

Metaobjects conforming to the meta::Type concept reflect base-level types. Models of this concept are also meta::Named and meta::Reversible. meta::Types reflecting native C++ types are not meta::ScopeMembers, reflections of all other types declared in source are meta::ScopeMembers. 

template <Object T> concept bool Type
 =  Named<T>
 && Reversible<T>
 && implementation-defined;

Type operations

Operations applicable to metaobjects conforming to the meta::Type concept: 

// get_reflected_ type
template <Type T>
struct get_reflected_type {
	using type = implementation-defined; 
};


template <Type T>
using get_reflected_type_t
 = typename get_reflected_type<T>::type;

The base-level type reflected by the meta::Type T.

TagType

Models of meta::TagType reflect the common aspects of elaborated types like classes, unions and enumerations. 

template <Object T> concept bool TagType
 =  Type<T>
 && Scope<T>
 && ScopeMember<T>
 && implementation-defined;

A meta::Specifier reflecting the elaborated type specifier (enum, union, struct or class) of the tag-type reflected by T.

Common aspects of base-level entities declared in a scope (the global scope, a namespace, a class, etc.) are reflected by metaobjects modelling the meta::ScopeMember concept. 

template <Object T> concept bool ScopeMember
 =  implementation-defined;

EnumMember

Enumerated type members (enumerators) are reflected by metaobjects modelling the meta::EnumMember concept. 

template <Object T> concept bool EnumMember
 =  ScopeMember<T>
 && implementation-defined;

RecordMember

Common aspects of structure, class and union members are reflected by metaobjects conforming to the meta::RecordMember concept. 

template <Object T> concept bool RecordMember
 =  ScopeMember<T>

Variable

Variables and class data members are reflected by meta::Variable metaobjects. 

template <Object T> concept bool Variable
 =  Named<T>

 && implementation-defined;

Constant

Compile-time constants (for example enumerators) are reflected by meta::Constant metaobjects. 

template <Object T> concept bool Constant
 =  Typed<T>
 && implementation-defined;

Specifier

template <Object T> concept bool Specifier
 =  Named<T>
 && implementation-defined;

 =  implementation-defined;
